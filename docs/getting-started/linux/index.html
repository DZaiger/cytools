Linux

Installation of CYTools is done using Docker. A Docker image allows us to package all of the dependencies and ensure that everything will work properly regardless of the Linux distribution. We have written convenient installation and launcher scripts so that you don't have to learn how use Docker or memorize long commands to use CYTools.

Requirements#

CYTools should work on almost any Linux distribution running on a modern x86-64 processor (with hardware virtualization enabled). The only requirement is being able to run Docker, and if you want to use the installation script you will also need to be able to run commands with sudo.

Installation instructions#

tip

For most users we recommend installing CYTools with a MOSEK license. MOSEK is an optimizer that is vastly superior to open-source alternatives. A license is required to use it, but an academic license can be requested for free at this link. MOSEK is not entirely necessary, but some KÃ¤hler cone computations may be slower or fail at large h1,1h^{1,1}h1,1 if it is not used.

  1. Install Docker Engine, if not already installed. Installation instructions can be found here.
  2. Download the source code of the latest stable release of CYTools from the following link, and extract the zip or tar.gz file into your desired location.

  3. (Optional) Activate MOSEK. This is done simply by placing the license file in the path [CYTools-root]/external/mosek/mosek.lic
  4. Navigate to the root directory of CYTools in your terminal, and run sudo make install . This will run a script that builds the Docker image and creates a shortcut in the start menu to start CYTools.

You can take a look at exactly what is being done by the install script by looking at the scripts/linux/install.sh (that's the point of open-source software!). In short, it builds the Docker image, copies a launcher script to /usr/local/bin/cytools and copies a couple of files to make the shortcut for the start menu.

Usage#

CYTools can be started simply by clicking on the icon that should appear on your start menu after running the installation script. Alternatively, it can be launched by running cytools in your terminal. It can be launched into bash mode with the -b flag, or mount a custom path with the -d [PATH] option. For more advanced options you can read our advanced usage instructions.

If you're still having issues, try reading through the advanced usage instructions and building the Docker image manually. Since there are a many different Linux distributions it is possible that our installation scripts don't always work. If this is the case, please let us know.
