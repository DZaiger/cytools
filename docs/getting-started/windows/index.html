Windows

Installation of CYTools is done using Docker. A Docker image allows us to package all of the dependencies and ensure that everything will work properly on Windows, even though most of the software was developed for Linux. We have written convenient installation and launcher scripts so that you don't have to learn how use Docker or memorize long commands to use CYTools.

Requirements#

CYTools should work on almost any Windows computer with Windows 10: Pro, Enterprise, or Education (Version 1903 or later); or Windows 10: Home (Version 2004 or later) running on a modern x86-64 processor (with hardware virtualization enabled). The only requirement is being able to run Docker (see the Docker installation guide for more details).

Installation instructions#

tip

For most users we recommend installing CYTools with a MOSEK license. MOSEK is an optimizer that is vastly superior to open-source alternatives. A license is required to use it, but an academic license can be requested for free at this link. MOSEK is not entirely necessary, but some KÃ¤hler cone computations may be slower or fail at large h1,1h^{1,1}h1,1 if it is not used.

  1. Install Docker Desktop. The installation guide for the Home version of Windows can be found here, and the guide for other versions can be found here.
  2. Download the source code of the latest stable release of CYTools from the following link, and extract the zip or tar.gz file into your desired location.

  3. (Optional) Activate MOSEK. This is done simply by placing the license file in the path [CYTools-root]/external/mosek/mosek.lic
  4. Navigate to [CYTools-root]/scripts/windows/ and double-click on the install.bat. This will run a script that builds the Docker image and creates a shortcut in the start menu to start CYTools. You may be prompted with a sign advising you against running the script since it is from an unknown publisher, but you can get around it by clicking on "More info" and then on "Run anyway".

You can take a look at exactly what is being done by the install script by opening the install.bat with a text editor (that's the point of open-source software!). In short, it builds the Docker image, copies a launcher script to %APPDATA%\CYTools (this usually corresponds to C:\Users/YOURUSERNAME\AppData\Roaming\CYTools\) and creates shortcuts on start menu and the Desktop.

Usage#

CYTools can be started simply by clicking on the icon that should appear on your start menu or on your desktop after running the installation script. For more advanced options you can read our advanced usage instructions.

Troubleshooting#

ErrorSolution
The install or launcher scripts fail to start or disappear.Some antivirus software may mark these scripts as suspicious and prevent them from running or even remove them. You can try making an exception for them for your antivirus, or you'll have to build and install CYTools without these scripts by following the advanced usage instructions.
Get https://registry-1.docker.io/v2/: dial tcp: lookup registry-1.docker.io on [::1]:53: read udp [::1]:57425->[::1]:53: read: connection refusedThis is likely because the Docker hub servers are overloaded and are refusing connections. All you have to do is try again a few times until it works.
Errors during the installation or start up.We have tried to print useful error messages when something goes wrong, so hopefully you can figure it out from there.

If you're still having issues, try reading through the advanced usage instructions and building the Docker image manually. Since we mainly focus on developing for Linux distributions it is possible that our installation scripts don't always work or that an update breaks them. If this is the case, please let us know.
